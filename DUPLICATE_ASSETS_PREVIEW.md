# 複製資產預覽

在導入資產後，下一步是學習如何使用 OpenUSD 的方法來複製資產。

## OpenUSD 複製資產的方法

OpenUSD 提供多種複製資產的方法，每種方法都有不同的用途和優勢：

### 1. 直接複製（Copy/Paste）

**用途**：創建資產的獨立副本

**方法**：
- 選擇場景中的資產
- 使用 `Ctrl+C` (Windows/Linux) 或 `Cmd+C` (Mac) 複製
- 使用 `Ctrl+V` (Windows/Linux) 或 `Cmd+V` (Mac) 貼上

**特點**：
- 創建完全獨立的副本
- 修改副本不會影響原始資產
- 每個副本都佔用獨立的記憶體

### 2. 實例化（Instancing）

**用途**：創建資產的實例，節省記憶體和提升效能

**方法**：
- 在 USD 中，使用 `instanceable` 屬性
- 通過應用程式 UI 創建實例

**特點**：
- 多個實例共享相同的幾何資料
- 節省記憶體（特別適合大量重複物件）
- 提升渲染效能
- 實例可以有不同的變換（位置、旋轉、縮放）

**適用場景**：
- 大量相同的物件（如貨架、箱子、設備）
- 需要高效能的場景
- 記憶體受限的環境

### 3. 引用（References）

**用途**：引用原始資產，保持與原始資產的連結

**方法**：
- 使用 USD 的 `reference` 機制
- 通過應用程式 UI 添加引用

**特點**：
- 保持與原始資產的連結
- 原始資產更新時，引用會自動更新
- 適合需要同步更新的場景

**適用場景**：
- 需要保持資產同步的場景
- 資產可能經常更新的情況
- 團隊協作場景

## 工作流程建議

### 選擇合適的方法

1. **少量資產**：使用直接複製
2. **大量重複資產**：使用實例化
3. **需要同步更新**：使用引用

### 效能考量

- **實例化**：最佳效能，適合大量重複物件
- **引用**：中等效能，適合需要同步的場景
- **直接複製**：較低效能，但最靈活
